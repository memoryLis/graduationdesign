# mybatis-plus

## mp@TableId注解

```java
eg：@TableId(value = "id", type = IdType.AUTO)
```

### 1. 数据库自增模式 (`IdType.AUTO`)

- **前提**：数据库表里的 `id` 字段必须勾选了 **“自动递增” (Auto Increment)**。
- **表现**：你往数据库插数据，不需要管 ID，数据库按 `1, 2, 3...` 顺序帮你生成。
- **注意**：你在代码里手动 `setId(99)` **没用**，会被数据库生成的数字覆盖掉。

### 2. 雪花算法模式 (`IdType.ASSIGN_ID`)

- **前提**：这是 MyBatis-Plus 的默认模式（或者显式配置）。
- **表现**：代码在执行插入前，会自动算出一个很长的数字（比如 `176543210987`）作为 ID。
- **注意**：无论数据库有没有勾选自增，都会优先用这个算出来的长数字。**微服务项目（比如你的商城）最推荐这个。**

### 3. 手动输入模式 (`IdType.INPUT`)

- **前提**：你在实体类上写了 `type = IdType.INPUT`。
- **表现**：MP 彻底不管 ID 了。
- **注意**：你必须在代码里 `setId(100)`，否则 SQL 报错（因为主键不能为空）。

### 4. 主键剥夺（IdType.AUTO + 手动设置）

- **你的操作**：在代码里写了 `address.setId(100L)`。
- **MP 的逻辑**：因为它看到你配置了 `AUTO`，它在生成 SQL 时会**直接无视**你对象里的 `100`。
- **结果**：
    1. 发送的 SQL 语句里**没有 ID 字段**。
    2. 数据库根据自己的计数器生成了一个新 ID（比如 `5`）。
    3. **最关键的一步**：MP 获取到数据库生成的 `5`，然后调用 `setId(5)` **强行覆盖**掉你原本的 `100`。
